핵심적 요소 : 노트를 확인하고 -> 입력하는 과정에서 생기는 집중력의 유지, 타격감 등 시각적 재미

목표 : 기존에 세워둔 기록보다 더 높은 점수로 갱신 또는 다른 플레이어와의 스코어 경쟁

확인 전 : 음악에 맞춘 노트의 생성, 도입부 재생, 플레이어의 준비(예열)

확인 : 집중력 극대화, 내려오는 순서를 기억함

입력 : 판정선과 노트간의 위치


입력 이펙트 : 입력했음을 각인시킴, 부가요소인 타격감 부여

판정 처리 : 판정선과의 간극을 기준으로 처리, 판정별 스코어 차이 발생

점수 변화 : 스코어 경쟁에서 패배 또는 승리, 기록 갱신을 했는지 못했는지




리듬게임 작동방식 예시

플레이어 
C1: 준비
C2: 노트의 확인
C3: 집중
C4: 판정 타이밍 계산
C5: 대기
C6: 입력
C7: 좋은 판정
C8: 보통 판정(콤보는 끊기지 않으나 좋은 판정에 비해 낮은 점수)
C9: 좋지않은 판정(콤보 끊김, 낮은 점수거나 0점)
C10: 반복
C11: 결과창 확인


노트
A1: 생성 타이밍
A2: 순서대로 생성(큐)
A3: 판정선 방향으로 이동
A4: 입력처리
A5: 위치값 반환
A6: 판정 출력
A7: 오브젝트를 파괴(일정높이▼ 또는 좋은 판정)




게임 컨셉 구상

첫번째 안 : 상,하,좌,우 네 방향에서 생성되는 노트, 판정선은 원형. 스마트폰에 적합
	--> 곡의 하이라이트 부분에선 분열, 혹은 합체 가능성 있음.

두번째 안 : 모델 게임은 유비트, 칸을 나눠서 노트를 순차적으로 생성, 스마트폰에 적합함. 판정은 판정시간 - 생성 후 지난시간
	--> 곡의 하이라이트 부분에서 생성속도를 빠르게 만듦.

세번째 안 : 판정선 화면 하단 직선, 나눈 칸 없이 노트를 뿌림. 가장 흔한 방식. 스마트폰에 적합함. 판정선이 확실하게 존재함.
